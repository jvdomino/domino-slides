Approaches

Vibe Coding vs
Agentic Engineering

Two Fundamentally Different Approaches to AI-Assisted Development

Vibe Coding

  • Developer describes intent in natural language
  • AI generates code in a conversational loop
  • Best for exploration, prototyping, and PoCs
  • Speed-first - idea to demo fast

Agentic Engineering

  • Detailed spec documents before writing code
  • Comprehensive unit, integration, and e2e testing
  • Multi-model code audits for quality assurance
  • Quality-first - production-grade from the start
Vibe coding is for demos and prototypes. Agentic engineering is for production-quality software. Claude Code supports both.

Context

CLAUDE.md

Persistent Project Context Loaded Every Session

  • What it is - A markdown file at your project root that tells Claude about your codebase, conventions, and workflows
  • Auto-loaded - Claude reads it automatically at the start of every session - no manual prompting needed
  • Bootstrap from repo - Run /init to auto-generate a CLAUDE.md with codebase documentation
Think of it as a README for your AI - the better the context, the better the output.
# CLAUDE.md ## Project ML scoring service - Python 3.11, FastAPI, scikit-learn. Deployed on Domino as a Model API. ## Repo layout src/ - application code src/models/ - trained model artifacts tests/ - pytest unit & integration tests domino.yaml - Domino environment config ## Conventions • Use ruff for linting & formatting • Type-hint all public functions • Tests must pass before committing ## Domino • Environment: py311-sklearn • Hardware tier: medium-k8s • Deploy via domino model publish

Agentic Pattern

The Ralph Loop

Autonomous, Self-Correcting Development Cycles in Claude Code

  • Originated by Geoffrey Huntley; available as an official Anthropic plugin (ralph-wiggum)
  • Core concept: a persistent loop that re-feeds Claude the same prompt, letting it iteratively improve until done
  • How it works: Claude works on task → tries to exit → Stop hook intercepts → re-injects prompt → repeat until completion
  • Philosophy: “Don’t aim for perfect on first try. Let the loop refine the work. Failures are predictable and informative.”

Usage

The Ralph Loop
Install & Best Practices

/plugin install ralph-wiggum@claude-plugins-official /ralph-loop:ralph-loop "Implement auth system with JWT, tests, and docs. Output <promise>DONE</promise>" --max-iterations 20 --completion-promise "DONE"
  • Always set --max-iterations as a safety net
  • Write prompts with binary success criteria
  • Run in git-tracked dirs; use Docker for AFK runs
  • Chain phases for complex projects
  • Review output like a PR - /cancel-ralph to stop

Configuration

Persisting Your
Claude Code Sessions

Authentication and session persistence on Domino

~

~/.claude → persistent dir: Replaces the local ~/.claude folder with a symlink to /mnt/.claude-users/<username>, preserving any existing contents

{}

~/.claude.json → persistent file: Symlinks ~/.claude.json (account metadata & OAuth) to /mnt/.claude-users/<username>.claude.json

ID

Per-user storage path: Uses $DOMINO_USER_NAME (falls back to $USER) to create a unique persistent directory at /mnt/.claude-users/<username>

Alternative Models

No Access to
Third-Party LLMs?

Domino 6.2 supports LLM hosting - use models like MiniMax M2.5 directly on-platform

# Point Claude Code to your Domino-hosted LLM export ANTHROPIC_BASE_URL="https://your-domino-host/model-endpoint" # Launch with your hosted model claude --model minimax/M2.5
  • ANTHROPIC_BASE_URL - set to your Domino-hosted LLM endpoint URL
  • --model - specify which hosted model to use
Claude Code works as the agentic framework - swap the underlying model to any compatible LLM hosted in Domino.
